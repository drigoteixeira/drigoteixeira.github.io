Design process

A structured design process helps me to take up any sort of problem, make sense of it and come up with solutions. Following is an outline of my design process.

Discovery

Goals, Users, Users' goals, Features, Competitors, Constraints and Research (survey results, customer feedback, analytics reports, user testing videos, etc).

Define

Insights (from the previous phase), definition about the users'goals. We'll build an use case diagram (for apps), site map and content inventory (for websites).

Prototype

We’ll decide the best way to organize information across the screens (sketching), how to navigate between screens and how the product would respond to user actions (wireframe and/or clickable prototype).

Design

I’ll flesh out the visuals of one or more key pages/screens using appropriate colors, typography, and icons. Once we agree on a visual style I’ll design the remaining screens in the same visual style.

Delivery

I’ll share the final deliverables in such a way that the development team can understand and implement. The final deliverables usually are: PNG, PDF and PSD formats and also a Style guide.

Iterate

Once the development team has built the product I’ll do a test run and check if the design has come up as we envisioned. I’ll write down the discrepancies and send it to the team for revision.
